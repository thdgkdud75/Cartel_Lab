# Design Rules (Impeccable)

## 🔥 Mandatory Reading (MUST)

Before doing ANY design-related task, you MUST read:

* .agents/skills
* .claude/skills
* .continue/skills

These directories contain design references, patterns, UI rules, and style guides.

DO NOT proceed without reading them.

---

## 🎯 Purpose

This project uses "impeccable" as a centralized design reference system.

All design decisions MUST follow the patterns defined in the skills directories.

---

## ⚠️ Core Rules

* Do NOT create new design styles arbitrarily
* Always follow existing UI/UX patterns from skills
* Maintain visual consistency across all components
* Reuse layout, spacing, and component patterns

---

## 🎨 Design Consistency

When designing:

* Match existing spacing, colors, and typography
* Follow previously used component structures
* Avoid introducing new styles unless explicitly required

---

## 🚫 Forbidden

* Ignoring skills directories
* Mixing inconsistent design styles
* Creating random UI patterns
* Overdesigning beyond existing system

---

## ✅ Expected Behavior

* Read → Understand → Reuse → Apply
* Keep design clean, consistent, and predictable

---

## ❗ Failure Condition

The task is considered FAILED if:

* Skills were not referenced
* Design is inconsistent with existing patterns
* New styles were introduced without reason

---

## 🔥 Summary

Always:

* Read skills
* Follow existing design
* Keep consistency
* Do not improvise styles
